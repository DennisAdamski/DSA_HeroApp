# iOS/Xcode Setup (SPM-first)

This repository uses a Swift Package Manager first workflow for iOS plugin dependencies.

## Prerequisites

- macOS with Xcode 15 or newer
- Flutter SDK installed and available in `PATH`
- Apple ID configured in Xcode (`Xcode > Settings > Accounts`)
- Physical iPad and USB cable for on-device testing (optional but recommended)

## Bootstrap Command

From repository root:

```bash
bash tool/ios_bootstrap_spm.sh
```

The script performs:

1. Host/tooling precondition checks (macOS, Xcode, Flutter, repo root)
2. Enables Flutter Swift Package Manager support
3. Runs `flutter clean` + `flutter pub get`
4. Runs `flutter build ios --config-only --simulator` to apply iOS migrations
5. Fails if `ios/Podfile` exists
6. Verifies SPM markers in `ios/Runner.xcodeproj/project.pbxproj`

## Expected File Changes After First Run on Mac

Commit the deterministic migration outputs generated by Flutter:

- `ios/Runner.xcodeproj/project.pbxproj`
- `ios/Runner.xcodeproj/xcshareddata/xcschemes/Runner.xcscheme` (if changed)

Do not commit generated ephemeral iOS files.

## Open in Xcode and Run

1. Open `ios/Runner.xcworkspace` in Xcode.
2. Select target `Runner`, then set:
   - a unique `Bundle Identifier` (for example `com.yourname.dsaheldenverwaltung`)
   - your Apple Team under `Signing & Capabilities`
   - `Automatically manage signing` enabled
3. Connect iPad, unlock it, and trust the computer.
4. Choose the iPad as the run target and run the app.
5. On first install with free Apple ID, trust the developer app certificate on the iPad:
   - `Settings > General > VPN & Device Management > Developer App > Trust`

## Troubleshooting

### Package resolution fails in Xcode

- Re-run:
  - `bash tool/ios_bootstrap_spm.sh`
- In Xcode, run `File > Packages > Reset Package Caches`.
- Ensure Xcode is at least version 15.

### Signing/provisioning errors

- Confirm Apple ID is added in Xcode Accounts.
- Ensure Bundle Identifier is unique.
- Re-select Team in `Signing & Capabilities`.
- For free Apple ID, device install provisioning is temporary and may need periodic reinstall.

### `ios/Podfile` appears unexpectedly

This repo is SPM-first. If `ios/Podfile` was added:

1. Remove `ios/Podfile` and any `Pods` artifacts.
2. Re-run `bash tool/ios_bootstrap_spm.sh`.
3. Verify SPM markers in `ios/Runner.xcodeproj/project.pbxproj`.
